# tools and data guidance

Use this reference when the task mentions google search, google analytics, google search console, wordpress, shopify, airtable, crm, or any connected source.

## source priority

1. user-provided files, brand guidelines, product catalog, and confirmed business data.
2. connected tools such as analytics, search console, airtable, crm, wordpress, or shopify.
3. live web/search results for current market, competitor, and destination research.
4. general best-practice reasoning with assumptions clearly labeled.

Do not invent connector data. If a connector is unavailable, say what data would be needed and provide a useful template or sample structure.

## google search use cases

Use search for:

- competitor pages and offers
- current destination information
- search result intent
- content gaps
- pricing references when appropriate
- seasonal demand or trend clues
- local attractions and practical travel facts

When using search results, separate facts from recommendations and cite sources when the chat environment supports citations.

## google analytics use cases

Use analytics data to answer:

- which channels generate traffic or bookings?
- which landing pages convert best or worst?
- where users drop off?
- which audiences, locations, or devices perform best?
- how paid campaigns compare with organic/social/email?

Useful fields/metrics: users, sessions, engaged sessions, engagement rate, conversions, revenue or booking events, source/medium, campaign, landing page, device, country/city, event names, conversion rate.

## google search console use cases

Use search console data to answer:

- which queries drive impressions and clicks?
- which pages have low ctr?
- which queries rank close to page one?
- which pages lost clicks or impressions?
- what new pages should be created?

Useful fields/metrics: query, page, clicks, impressions, ctr, position, country, device, date range.

## wordpress guidance

For wordpress content or publishing tasks:

- produce clean title, slug, meta title, meta description, excerpt, headings, body content, faqs, image alt text, categories, tags, and internal links.
- do not publish live unless the user explicitly asks and the tool is available.
- preserve brand voice and formatting.
- recommend plugins or implementation only as suggestions unless the user confirms their stack.

## shopify guidance

For shopify tourism products or trip packages:

- treat trips as products/services with title, description, variants, availability, price/from, images, collection, tags, seo title, seo description, and faq.
- variants may include date, group/private, hotel level, number of travelers, or add-ons.
- ensure booking/payment/cancellation details are visible.

## airtable and crm guidance

For airtable or crm tasks:

- identify tables/objects such as trips, leads, customers, bookings, campaigns, tasks, and content calendar.
- suggest fields, statuses, views, automations, and follow-up rules.
- for sales analysis, segment by source, destination, status, response time, booking rate, and revenue.

## minimum data needed by task

- seo plan: domain, target locations, target services, current pages, competitors, search console if available.
- landing page: offer, audience, destination, price/from, cta, proof, form/whatsapp flow.
- campaign: objective, audience, budget if available, dates, offer, channel, landing page.
- sales script: product, persona, objections, booking process, payment terms.
- content calendar: destinations, season, channels, frequency, offers, language.
